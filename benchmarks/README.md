# CompressedLUT CPU/CUDA Benchmarks

This directory contains reusable software benchmarks for the compressed evaluator generated by the CompressedLUT algorithm.

## Build

```bash
make bench_cpu
make bench_cuda
```

The CUDA target defaults to `sm_120` for the RTX PRO 6000 Blackwell GPUs available on this server.

## Run

Quick smoke run:

```bash
/home/luo00466/miniconda3/envs/py310/bin/python benchmarks/run_benchmarks.py --quick --out bench_results/quick
```

Full run on GPU 7:

```bash
/home/luo00466/miniconda3/envs/py310/bin/python benchmarks/run_benchmarks.py --device 7 --out bench_results/latest
```

The runner writes `cpu.csv`, `cuda.csv`, `all_results.csv`, `summary.md`, and `throughput_sequential.png`.

## Profile CUDA Kernels

Focused Nsight Systems and Nsight Compute profiles:

```bash
DATASET=example_txt PATTERN=random VARIANTS="plain_lut compresslut" \
  benchmarks/profile_cuda.sh
```

Useful environment overrides:

- `OUT_DIR=bench_results/profile`
- `DEVICE=7`
- `LOOKUPS=16777216`
- `REPEATS=1`
- `NCU_SET=basic` (`detailed` is slower but gives richer memory/scheduler/source-counter sections)
- `DATASET=example_txt|exp_x_minus_1|sigmoid_8x|sqrt_x|sin_half_pi_x|log1p_x|tanh_3x`
- `PATTERN=sequential|random`
- `VARIANTS="plain_lut compresslut cuda_math_f64"`

The script emits one CSV, one `.nsys-rep`, and one `.ncu-rep` per selected variant. Nsight Systems is best for launch/runtime timeline and memory-copy context; Nsight Compute is best for SM occupancy, memory hierarchy pressure, achieved bandwidth, warp stalls, and instruction mix.

To sweep multiple LUTs:

```bash
DATASETS="example_txt exp_x_minus_1 sigmoid_8x" PATTERNS=random \
  benchmarks/profile_all_cuda.sh
```

## Variants

- `plain_lut`: direct table lookup baseline.
- `compresslut`: metadata-driven decode of the compressed LUT.
- `libm`: CPU math-library evaluation for generated function tables.
- `cuda_math_f64`: CUDA double-precision math evaluation for generated function tables.

Throughput timings exclude compression construction and host-device copies. Compression time and compressed bit counts are reported in the CSV columns.
